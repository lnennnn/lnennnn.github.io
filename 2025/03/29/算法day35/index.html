0-1背包 二维

建立一个n行（volume+1）列的dp数组，dp[i][j]表示当背包容量为j时放入第二个物品可以得到的最大价值

对于每个物品可以选择放or不放

  • 放-> dp[i][j] = dp[i-1][j-weight[i]] + value[i]
  • 不放-> dp[i][j] = dp[i-1][j]

分割等和子集

回溯法（应该没错吧）但超时

动态规划：

可以抽象为一个背包问题，由于数组的和我们是知道的，所以等价于当背包容量为 sum(nums)/2 时，用数组中的元素是否能将这个背包装满

有人会疑惑， 0-1 背包问题求的是最大值，而子集求的是等于某个值，这两个不是一回事啊？

但其实是一回事，物品的重量和物品的价值是一样的，所选物品的重量只能小于等于背包重量，因此所选物品的价值也只能小于等于背包的重量，所以求的是所选物品价值的最大值，当物品价值的最大值等于背包重量时，其实也就表示所选物品的重量等于背包价值，也就是找到了子集和等于目标值。

最后一块石头的重量

题目可以转化为，将一个数组分成两个子集，使这两个子集的和之差最小

目标和

  1. 含义：dp[i][j]：从下标为[0…i]的物品里任取，可以填满j这么⼤容积的包的方法个数

  2. 递推式：dp[i][j] = dp[i-1][j] + dp[i-1][j-nums[i]]

    dp[i-1][j]是不将物品i放入背包的方式数

    dp[i-1][j-nums[i]]是将物品i放入背包的方式数

  3. 初始化：dp[0][0]= 1 表示装满容量为0的背包，有1种⽅法，就是装0件物品。
    如果nums[0]在范围内的话，dp[0][nums[0]] = 1
    ，其他全为0

  4. 计算顺序：顺序，行优先

一和零

这道题的背包有两个维度，即背包有两个维度的容量要求

  • dp数组定义：dp[i][j]表示当背包（子集）可容纳i个0和j个1时，子集的最多元素个数
  • dp数组初始化：第一列和第一行均初始化为0
  • 遍历顺序：先遍历物品（字符串数组中的元素），再遍历背包，背包遍历倒序

总结一下

0-1背包递归式的统一特征就是，在写递推式的时候分为取（dp[i][j]） 和 不取（dp[i-x]，x一般为当前物品的重量，或者说是对背包的占有值） 两种情况进行讨论。

dp数组的下标与背包的容量要求有关，dp数组中每一个的值与我们要求的值有关，包括：

  • 最大价值（纯0-1背包）
  • 满足条件情况个数（目标和）
  • 最多元素个数（一和零）
  • 是否能够填满背包（分割等和子集，遇到满足条件的即返回True）。

有些对于背包的容量要求需要我们自己去定义，比如:

  • 分割等和子集 target_sum = sum(nums) // 2
  • 最后一块石头重量以及目标和 target_sum = sum(nums) // 2
  • 目标和 target_sum = (target + sum(nums)) // 2

这几个题的相似点是，都需要将某个集合分成两堆

最后关于遍历顺序，一般是先遍历物品，再遍历背包，背包遍历倒序
