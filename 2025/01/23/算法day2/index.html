长度最小子数组

题目描述： 给定一个含有 n 个正整数的数组和一个正整数 target 。找出该数组中满足其总和大于等于 target 的长度最小的
子数组 [numsl, numsl+1, ..., numsr-1, numsr] ，并返回其长度。如果不存在符合条件的子数组，返回 0 。
题目链接：https://leetcode.cn/problems/minimum-size-subarray-sum/
文章讲解：https://programmercarl.com/0209.%E9%95%BF%E5%BA%A6%E6%9C%80%E5%B0%8F%E7%9A%84%E5%AD%90%E6%95%B0%E7%BB%84.html
视频讲解：https://www.bilibili.com/video/BV1tZ4y1q7XE

（1）暴力解法

暴力解法就是用两个循环遍历得到所有的子数组，子数组起始位置 i 从 0 到 len(nums) - 1 ，终止位置 j 从 i 到 len(nums) - 1 ，计算每个子数组长度，找出满足其总和大于等于 target 的长度最小的所有子数组并返回其中长度最小子数组的长度，时间复杂度为 O(n^2) 。完整代码如下：

（2）滑动窗口

滑动窗口，就是不断的调节子序列的起始位置和终止位置，从而得出我们要想的结果。
在本题中实现滑动窗口，主要确定如下三点：

  • 窗口内是什么？
    窗口就是满足其和≥s的长度最小的连续子数组。
  • 如何移动窗口的起始位置？
    如果当前窗口的值大于等于s了，窗口就要向前移动了（也就是该缩小了）。
  • 如何移动窗口的结束位置？
    窗口的结束位置就是遍历数组的指针，也就是for循环里的索引。

滑动窗口的精妙之处在于根据当前子序列和大小的情况，不断调节子序列的起始位置从而将 O(n^2) 暴力解法降为 O(n) 。

不要以为for里放一个while就以为是 O(n^2) ， 主要是看每一个元素被操作的次数，每个元素在滑动窗后进来操作一次，出去操作一次，每个元素都是被操作两次，所以时间复杂度是 2 × n 也就是 O(n) 。

螺旋矩阵

题目描述： 给你一个正整数 n ，生成一个包含 1 到 n^2 所有元素，且元素按顺时针顺序螺旋排列的 n x n 正方形矩阵 matrix 。
题目链接：https://leetcode.cn/problems/spiral-matrix-ii/
文章讲解：https://programmercarl.com/0059.%E8%9E%BA%E6%97%8B%E7%9F%A9%E9%98%B5II.html
视频讲解：https://www.bilibili.com/video/BV1SL4y1N7mV/

首先，需要明确需要转几圈。对于正整数n，易得当n为偶数时圈数为n//2，当n为奇数时除了中间那一个圈数也为n//2。所以第一层循环的次数应该为n//2，且对于n为奇数的情况需要最后单独给中间那个数赋值。

其次，需要明确每一圈内需要干什么。每一圈需要遍历四条边，为了不出错，我们需要以同一个逻辑处理每一条边，这里用左闭右开比较好，即每一条边仅处理第一个元素，最后一个元素作为下一条边的起始元素。对于不同的圈，需要变化的是第一个点的起始位置 (startx, starty) 以及缩进的偏移 offset ，所以需要在第一层循环的结尾对这几个值进行更新。

前缀和

题目描述： 给定一个整数数组 Array，请计算该数组在每个指定区间内元素的总和。
题目链接：https://kamacoder.com/problempage.php?pid=1070

前缀和的思想是重复利用计算过的子数组之和，从而降低区间查询需要累加计算的次数。我们先做累加，即 pre[i] 表示下标 0 到 i 的 arr[i] 累加之和。而 pre 数组是我们之前就计算好的累加和，所以后面每次求区间和的之后 我们只需要 O(1) 的操作。

特别注意： 在使用前缀和求解的时候，要特别注意求解区间。
eg.如果我们要求区间下标 [2, 5] 的区间和，那么应该是 p[5] - p[1]，而不是 p[5] - p[2]。
